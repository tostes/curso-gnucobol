## Sobre o Projeto (PT-BR)

Este projeto é uma iniciativa de pesquisa e desenvolvimento dedicada a explorar a robustez do **GnuCOBOL** aplicado a sistemas de alta disponibilidade. O objetivo central é recriar a arquitetura lógica de uma plataforma governamental (inspirada no **ReBEC** - Registro Brasileiro de Ensaios Clínicos), utilizando uma linguagem de baixo nível e alta performance para processamento de transações.

### Por que COBOL?
Como Senior Software Architect e Data Engineer, acredito que entender os fundamentos da computação transacional é essencial. Este projeto não busca apenas replicar funcionalidades, mas sim aplicar princípios modernos de arquitetura de software (modularização, separação de camadas, integração com bancos relacionais) em uma linguagem que definiu a era do processamento de dados mundial.

### Objetivos
*   **Estudo Prático:** Aprender a sintaxe e as capacidades do GnuCOBOL 3.2.
*   **Integração de Sistemas:** Conectar o GnuCOBOL a um banco de dados **PostgreSQL** utilizando `GixSQL`.
*   **Interface de Alta Performance:** Implementar uma TUI (Terminal User Interface) robusta, utilizando `ncurses` e `SCREEN SECTION` para replicar a experiência de sistemas bancários de alta performance.
*   **Aprendizado contínuo:** Aplicar boas práticas de engenharia de dados em um ambiente de desenvolvimento controlado no Linux.

## About the Project (EN)

This project is a research and development initiative dedicated to exploring the robustness of **GnuCOBOL** in high-availability environments. The core goal is to recreate the logical architecture of a government platform (inspired by **ReBEC** - Brazilian Registry of Clinical Trials) using a low-level, high-performance language for transaction processing.

### Why COBOL?
As a Senior Software Architect and Data Engineer, I believe that understanding the foundations of transactional computing is essential. This project is not just about replicating functionality, but about applying modern software architecture principles (modularization, layer separation, relational database integration) in a language that defined the era of global data processing.

### Objectives
*   **Hands-on Learning:** Master GnuCOBOL 3.2 syntax and capabilities.
*   **Systems Integration:** Connect GnuCOBOL to a **PostgreSQL** database using `GixSQL`.
*   **High-Performance Interface:** Implement a robust TUI (Terminal User Interface) using `ncurses` and `SCREEN SECTION` to replicate the experience of high-performance legacy banking systems.
*   **Continuous Improvement:** Apply data engineering best practices in a controlled Linux development environment.
